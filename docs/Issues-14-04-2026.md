## 🔴 Fix #1 — Create Portfolio returns 401 (Critical)

### Root Cause (confirmed via deep inspection)

The frontend is doing everything correctly:

* Sends `POST https://api.propequitylab.com/portfolios`
* Includes `Authorization: Bearer eyJ...` (a valid Clerk JWT, 771 chars, not expired)
* Sends body `{"name":"My Portfolio","type":"actual"}`
* Uses `axios` (aliased as `kt`/`Et`) with a request interceptor

The JWT payload contains:

json

```json
{
"iss":"https://clerk.propequitylab.com",
"aud":"propequity-api",
"azp":"https://www.propequitylab.com"
}
```

The backend responds `{"detail": "Could not validate credentials"}` — this is a FastAPI/python-jose error message that fires when JWT signature verification fails. The token is valid and well-formed, so the problem is on the  **backend side** : it is trying to verify the JWT using the wrong key or wrong audience string.

### Most Likely Causes (in order of probability)

**A. Wrong JWKS URL on the backend** — the backend is fetching Clerk's public keys from a dev/test Clerk instance URL instead of the production one (`https://clerk.propequitylab.com/.well-known/jwks.json`).

**B. Audience mismatch** — the backend's `algorithms` or `audience` config doesn't match `"propequity-api"`.

**C. Dev key used in production** — the frontend uses `pk_live_...` (production Clerk), but the backend might be configured with a `pk_test_...` secret key.

### Backend Fix (FastAPI example)

python

```python
# auth.py — wherever your JWT verification lives
from jose import jwt, JWTError
import httpx

CLERK_JWKS_URL ="https://clerk.propequitylab.com/.well-known/jwks.json"
CLERK_AUDIENCE ="propequity-api"# Must match the "aud" claim in the JWT
CLERK_ISSUER   ="https://clerk.propequitylab.com"# Must match "iss" claim

asyncdefget_current_user(token:str= Depends(oauth2_scheme)):
# Fetch the public JWKS from Clerk (cache this — don't fetch on every request)
asyncwith httpx.AsyncClient()as client:
        jwks =(await client.get(CLERK_JWKS_URL)).json()

try:
        payload = jwt.decode(
            token,
            jwks,# Pass the full JWKS object
            algorithms=["RS256"],
            audience=CLERK_AUDIENCE,
            issuer=CLERK_ISSUER,
            options={"leeway":10}# 10s clock-skew tolerance
)
return payload
except JWTError as e:
raise HTTPException(status_code=401, detail="Could not validate credentials")
```

**Important:** Cache the JWKS (e.g. with a TTL of 1 hour) using something like `cachetools` — fetching it on every request is a performance bug and will get rate-limited.

If you're using a library like `clerk-backend-api` for Python, ensure your `secret_key` in the environment is the **live** secret key (`sk_live_...`), not a test key.

## 🔴 Fix #2 — Add a Portfolio Creation Form/Modal (Critical)

### Root Cause

The "Create Portfolio" button fires `createPortfolio({name:"My Portfolio", type:"actual"})` with hardcoded values immediately on click — there is no user input step. Even if the auth bug is fixed, users get no control over naming or configuring their portfolio.

### Frontend Fix

Replace the direct API call in the click handler with a modal. Here's the pattern:

tsx

```tsx
// CreatePortfolioModal.tsx
import{ useState }from"react";
import{ toast }from"sonner";
import{ apiService }from"@/services/api";

exportfunctionCreatePortfolioModal({ onSuccess }:{onSuccess:()=>void}){
const[open, setOpen]=useState(false);
const[name, setName]=useState("");
const[type, setType]=useState<"actual"|"investment">("actual");
const[loading, setLoading]=useState(false);

consthandleCreate=async()=>{
if(!name.trim()){
      toast.error("Please enter a portfolio name");
return;
}
setLoading(true);
try{
await apiService.createPortfolio({ name: name.trim(), type });
      toast.success("Portfolio created!");
setOpen(false);
onSuccess();
}catch(err){
      toast.error("Failed to create portfolio. Please try again.");
}finally{
setLoading(false);
}
};

return(
<>
<buttononClick={()=>setOpen(true)}className="btn-primary">
        + Create Portfolio
</button>

{open &&(
<divclassName="modal-overlay">
<divclassName="modal">
<h2>Create Your Portfolio</h2>
<label>Portfolio Name</label>
<input
type="text"
placeholder="e.g. My Investment Portfolio"
value={name}
onChange={e =>setName(e.target.value)}
maxLength={50}
/>
<label>Type</label>
<selectvalue={type}onChange={e =>setType(e.target.valueasany)}>
<optionvalue="actual">Actual (track real properties)</option>
<optionvalue="investment">Investment (plan future purchases)</option>
</select>
<divclassName="modal-actions">
<buttononClick={()=>setOpen(false)}disabled={loading}>Cancel</button>
<buttononClick={handleCreate}disabled={loading ||!name.trim()}>
{loading ?"Creating...":"Create Portfolio"}
</button>
</div>
</div>
</div>
)}
</>
);
}
```

The key changes from the current behaviour: (1) the button opens the modal instead of firing the API directly; (2) the user names the portfolio; (3) loading/disabled state prevents double-submit; (4) the error message is user-friendly.

## 🟠 Fix #3 — Progress & Plans pages show infinite spinner (High)

### Root Cause

The pages render a loading spinner while waiting for portfolio data to load via their data hook. When there is no portfolio (the fetch returns empty or 401), the hook never resolves to a `!isLoading` state — it stays stuck. There is no guard that checks "if no portfolio exists, show an empty state" before entering the loading branch.

Projections correctly handles this with an empty state: `"Please create a portfolio first."` — that pattern needs to be replicated.

### Frontend Fix

Add a portfolio existence check at the top of each page component. The app already has a `currentPortfolio` value in state (visible via the sidebar "No Portfolio" text):

tsx

```tsx
// ProgressPage.tsx  (same pattern for PlansPage.tsx)
import{ usePortfolio }from"@/hooks/usePortfolio";
import{EmptyPortfolioState}from"@/components/EmptyPortfolioState";

exportfunctionProgressPage(){
const{ portfolio, isLoading: portfolioLoading }=usePortfolio();

// Guard: if portfolio hasn't loaded yet, show a brief skeleton
if(portfolioLoading){
return<PageSkeleton/>;
}

// Guard: if no portfolio exists, show the empty state CTA (NOT a spinner)
if(!portfolio){
return<EmptyPortfolioState/>;// ← same component already used on Projections
}

// ... rest of page
}
```

The `EmptyPortfolioState` component already exists in the codebase (it's used on Projections). Reusing it makes this a 2-line change per page. Also fix the page title — the `<h1>` (or `document.title`) should reflect the current route, not always say "Dashboard".

## 🟠 Fix #4 — Error toast is too brief and message is too technical (High)

### Root Cause

The app uses **Sonner** for toast notifications. The default Sonner `toast.error()` duration is 4 seconds, and the message text is the raw API error: `"Could not validate credentials"`. Both need to change.

### Frontend Fix

**A. Make the error message user-friendly** — change it at the call site in the `CreatePortfolio` handler:

tsx

```tsx
// Before (current):
toast.error("Failed to create portfolio",{
  description: error?.response?.data?.detail // "Could not validate credentials"
});

// After (fix):
toast.error("Couldn't create your portfolio",{
  description:"Something went wrong on our end. Please try again, or contact support if it persists.",
  duration:8000,// 8 seconds — enough time to read and act
  action:{
    label:"Contact Support",
onClick:()=>window.location.href="mailto:support@propequitylab.com"
}
});
```

**B. Set a longer default duration globally** — in your `App.tsx` or root layout where `<Toaster />` is rendered:

tsx

```tsx
// App.tsx
import{Toaster}from"sonner";

<Toaster
position="bottom-right"
toastOptions={{
    duration:5000,// 5s default for all toasts
    classNames:{
      error:"toast-error",// style to make it more visible
}
}}
/>
```

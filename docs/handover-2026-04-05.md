## Bug Fixes Applied

### Bug 1 — `annualize_amount` silently returned 0 for lowercase frequencies
**File:** `backend/utils/calculations.py`  
DB stores frequencies as lowercase (`"monthly"`, `"weekly"`). `FREQUENCY_MULTIPLIERS`
only had Title Case keys (`"Monthly"`, `"Weekly"`). Lookup missed every time → 0.  
**Fix:** Added all lowercase variants to the dict.

### Bug 2 — Income and expenses not annualised; rental income not annualised
**File:** `backend/routes/portfolios.py`  
Route was summing raw `amount` values without multiplying by frequency. A monthly
salary of $5,000 was reported as $5,000/year instead of $60,000/year.  
**Fix:** Wrapped all income/expense sums with `annualize_amount()`. Rental income
uses `"weekly"` as default frequency (AU convention — matches `RentalDetails` model default).

### Bug 3 — `total_value` excluded assets; `total_equity` calculation was wrong
**File:** `backend/routes/portfolios.py`  
`total_value` was only property value. `total_equity` used `total_property_equity`
(property value minus debt) rather than `total_value - total_property_debt`.  
**Fix:**
```python
total_value   = total_property_value + total_assets       # full portfolio value
total_equity  = total_value - total_property_debt         # equity across everything
net_worth     = total_equity - total_liabilities          # minus non-property liabilities
```

### Bug 4 — `goal_year` was read from `target_year` (doesn't exist) instead of computing it
**File:** `backend/routes/portfolios.py`  
`goal_settings` stores `target_retirement_age` (integer), not a precomputed year.
Code did `goal_settings.get("target_year")` → always `None`.  
**Fix:** Added `_compute_goal_year(goal_settings, date_of_birth)` helper:
```python
def _compute_goal_year(goal_settings, date_of_birth):
    target_age = goal_settings.get("target_retirement_age")
    birth_year = int(date_of_birth[:4])
    current_year = date.today().year
    current_age = current_year - birth_year
    return str(current_year + (int(target_age) - current_age))
```

## Tests Added

### `backend/tests/test_portfolio_summary.py` (new file — 20 tests)
Direct handler tests using SQLite in-memory + StaticPool. Calls `get_portfolio_summary`
route handler directly (bypasses FastAPI TestClient — incompatible with Starlette 1.0.0).

Test classes:
- `TestPortfolioSummaryShape` — correct field names present, forbidden old names absent
- `TestPortfolioSummaryValues` — exact Decimal values for every field in known seed data
- `TestPortfolioSummaryIsolation` — other user's portfolio_id → 404

Seed data used in value tests:
```
Property:  value=500k, loan=400k, rental=2000/week
Income:    5000/month
Expense:   1000/month
Asset:     10k
Liability: 3k
```
Expected output: total_value=510k, total_debt=400k, total_equity=110k, net_worth=107k,
annual_income=164k (104k rental + 60k salary), annual_expenses=12k, annual_cashflow=152k.

### `backend/tests/test_calculations.py` (extended)
Added `TestAnnualizeAmountLowercaseFrequency` with 6 tests covering `monthly`, `weekly`,
`fortnightly`, `annual`, `Annually` (Title Case), `quarterly`.

### `backend/tests/conftest.py` (new file)
Stubs out the `routes` package to prevent `routes/__init__.py` from executing at import
time. That file instantiates `APIRouter()` which crashes under Starlette 1.0.0 (removed
`on_startup` kwarg). Stub lets route submodules be imported explicitly while blocking
the package-level init.
